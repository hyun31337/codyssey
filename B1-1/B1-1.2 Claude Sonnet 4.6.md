## 1. 취약점 핵심 요약 (경영진 보고용)

> XZ Utils 압축 라이브러리의 특정 버전(5.6.0~5.6.1)에 **공급망 공격(Supply Chain Attack)** 을 통해 악성 코드가 삽입되었으며, 이를 통해 **인증 없이 원격으로 시스템 전체를 장악**할 수 있는 백도어가 확인되었습니다. CVSS 최고 등급(10.0)으로, 해당 버전 사용 시 **즉각적인 조치**가 필요합니다.

---

## 2. 기술적 위협 요인 및 공격 시나리오

### 2-1. 취약점 본질
| 항목 | 내용 |
|------|------|
| **CVE ID** | CVE-2024-3094 |
| **취약점 유형** | CWE-506 – Embedded Malicious Code (악성 코드 내장) |
| **CVSS 점수** | **10.0 / Critical** (NVD 기준 최고 등급) |
| **공격 벡터** | 네트워크(원격) / 인증 불필요 |
| **영향 범위** | XZ Utils 5.6.0, 5.6.1이 설치된 리눅스 시스템 |

### 2-2. 공격 시나리오

```
[공격자]
    │
    ▼
① XZ Utils 5.6.0/5.6.1 내 악성 코드 존재 확인
    │
    ▼
② 해당 라이브러리가 systemd를 통해 OpenSSH에 로드됨
    │
    ▼
③ 공격자는 특수 제작된 SSH 요청 전송
    │
    ▼
④ 인증 우회 → 원격 코드 실행(RCE) 달성
    │
    ▼
⑤ 시스템 전체 장악 (root 권한 수준)
```

> **핵심 위협 포인트:**
> - 이 악성 코드는 **외부 기여자에 의해 의도적으로 삽입**된 공급망 공격입니다.
> - 단순 소프트웨어 버그가 아닌 **지능형 지속 위협(APT)** 수준의 공격 방식입니다.
> - SSH 데몬이 동작 중이고 해당 라이브러리가 로드된 환경에서만 공격이 실현됩니다.

## 4. 조치 및 대응 프로세스 (Action Items)

### Phase 1 — 즉시 조치 (24시간 이내)

```bash
# Step 1. XZ Utils 버전 확인 (전체 서버 대상)
xz --version

# Step 2. 패키지 관리자를 통한 버전 확인 (Debian/Ubuntu 계열)
dpkg -l | grep xz-utils

# Step 3. 패키지 관리자를 통한 버전 확인 (RHEL/Fedora 계열)
rpm -qa | grep xz
```

> 5.6.0 또는 5.6.1 확인 시 → **즉시 Phase 2로 이행**

---

### Phase 2 — 취약 버전 확인 시 조치

```bash
# [Debian/Ubuntu] 안전 버전으로 다운그레이드
sudo apt install xz-utils=5.4.1-0.2  # 배포판별 가용 버전 확인 후 적용

# [Fedora] 다운그레이드
sudo dnf downgrade xz

# 조치 후 SSH 데몬 재시작
sudo systemctl restart sshd

# 조치 후 버전 재확인
xz --version
```
