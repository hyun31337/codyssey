# CVE-2024-3094 취약점 요약 보고서

## 0. 위험도 산정 전 확인 질문

우리 인프라의 실제 영향도를 확정하려면 아래 정보가 필요합니다.

1. 운영 중인 리눅스 서버의 배포판 및 버전은 무엇입니까?  
   - Debian, Fedora, Ubuntu 등 최신 테스트/불안정 빌드 사용 여부 확인 필요
2. 각 서버에 설치된 **XZ Utils 버전**은 무엇입니까?  
   - 영향 버전: **5.6.0, 5.6.1**
3. SSH 서비스가 운영 중이며 외부 또는 내부망에서 접근 가능한 상태입니까?  
   - 본 취약점은 제공된 정보 기준 **SSH 원격 코드 실행**과 관련 있음

---

## 1. 취약점 핵심 요약  
### 비전문가 / 경영진 보고용 1줄

**XZ Utils 5.6.0 및 5.6.1에 악성 백도어가 포함되어 특정 조건에서 인증되지 않은 SSH 원격 코드 실행이 가능하므로, 해당 버전 사용 여부를 즉시 확인하고 안전 버전으로 조치해야 합니다.**

---

## 2. 기술적 위협 요인 및 공격 시나리오

### 2.1 취약점 개요

| 항목 | 내용 |
|---|---|
| CVE ID | CVE-2024-3094 |
| 취약 소프트웨어 | XZ Utils |
| 영향 버전 | 5.6.0, 5.6.1 |
| 취약점 유형 | CWE-506: Embedded Malicious Code |
| CVSS 점수 | 10.0 / Critical |
| 주요 영향 | 특정 조건에서 인증되지 않은 SSH 원격 코드 실행 가능 |
| 영향 시스템 | 주로 최신 테스트/불안정 빌드의 Debian, Fedora, Ubuntu 등 리눅스 배포판 |

### 2.2 기술적 위협 요인

- XZ Utils 특정 버전에 **악성 코드가 내장**되어 있음
- 일반적인 설정 오류나 단순 버그가 아니라 **소프트웨어 공급망 내 악성 코드 삽입 문제**
- 제공된 정보 기준, 특정 조건에서 **SSH를 통한 비인가 원격 코드 실행**으로 이어질 수 있음
- CVSS 10.0으로 평가된 **Critical 등급 취약점**

### 2.3 공격 시나리오

1. 공격 대상 시스템에 XZ Utils **5.6.0 또는 5.6.1**이 설치되어 있음
2. 해당 시스템이 취약점이 동작 가능한 조건을 충족함
3. 공격자가 SSH 경로를 통해 취약 조건을 악용
4. 인증되지 않은 상태에서 원격 코드 실행 가능
5. 결과적으로 시스템 내 명령 실행 또는 추가 침해로 이어질 수 있음

> 단, 실제 악용 가능 여부는 **XZ Utils 버전, 리눅스 배포판, 빌드 상태, SSH 사용 여부** 확인 후 판단해야 합니다.

## 4. 조치 및 대응 프로세스

## Phase 1. 즉시 점검

전체 리눅스 서버를 대상으로 XZ Utils 버전을 확인합니다.

```bash
xz --version
```

Debian / Ubuntu 계열:

```bash
dpkg -l | grep xz-utils
```

Fedora / RPM 계열:

```bash
rpm -qa | grep xz
```

확인 결과가 아래 버전이면 즉시 조치합니다.

- XZ Utils 5.6.0
- XZ Utils 5.6.1

---

## Phase 2. 취약 버전 조치

취약 버전이 확인된 경우 다음 중 하나를 수행합니다.

1. **XZ Utils를 5.4.x 등 안전한 버전으로 다운그레이드**
2. 배포판에서 제공하는 **최신 패치 버전 적용**
3. 조치 후 버전 재확인

예시:

```bash
xz --version
```

조치 후 출력 결과가 5.6.0 또는 5.6.1이 아니어야 합니다.

조치 완료 후 다음을 확인합니다.

1. 전체 서버에서 XZ Utils 5.6.0 / 5.6.1이 제거되었는지 확인
2. 패치 또는 다운그레이드가 정상 적용되었는지 확인
