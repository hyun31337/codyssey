## 1. 경영진 요약 (Executive Summary)



> 리눅스 시스템에 광범위하게 사용되는 데이터 압축 라이브러리(XZ Utils)에 **공급망 공격(Supply Chain Attack)** 방식으로 악성코드가 삽입되어, 외부 공격자가 **인증 없이 서버에 원격 접속 및 명령 실행**이 가능한 최고 수준의 취약점이 확인됨.



---



## 2. 기술적 위협 요인 및 공격 시나리오



### 2-1. 취약점 본질

- **유형:** CWE-506 — 소프트웨어 빌드 과정에 악의적 코드가 의도적으로 삽입된 **공급망 침해(Supply Chain Compromise)**

- **삽입 경로:** 오픈소스 기여자(악의적 행위자)가 장기간 신뢰를 구축한 뒤 XZ Utils 공식 소스코드에 백도어 코드를 직접 커밋

- **작동 방식:**

  - 악성 코드가 `liblzma` 라이브러리에 삽입됨

  - `systemd`를 통해 `sshd(OpenSSH 데몬)`와 연동되는 환경에서 활성화

  - 공격자가 **특수 제작된 SSH 패킷**을 전송하여 인증 절차를 완전 우회

  - 인증 없이 **원격 코드 실행(RCE)** 달성 가능



### 2-2. 공격 시나리오 (확인된 사실 기반)



```

[외부 공격자]

      │

      │  특수 제작 SSH 패킷 전송

      ▼

[인터넷 노출 SSH 포트 (22/tcp)]

      │

      │  백도어 활성화 (liblzma 후킹)

      ▼

[SSH 인증 우회]

      │

      │  인증 없이 명령 실행

      ▼

[서버 완전 장악 (RCE)]

```



- **전제 조건:**

  - XZ Utils 5.6.0 또는 5.6.1이 설치된 시스템

  - `systemd` 환경에서 `sshd`가 동작 중인 리눅스 서버

  - 해당 SSH 포트가 외부에 노출된 경우 즉각적 위협

## 3. 영향도 판단 기준



인프라 담당자는 아래 기준에 따라 **자체 영향도를 즉시 판단**할 것.



| 판단 항목 | 확인 명령어 | 위험 판정 기준 |

|---|---|---|

| XZ Utils 버전 확인 | `xz --version` | **5.6.0 또는 5.6.1 → 즉시 조치** |

| 설치 패키지 확인 | `rpm -q xz` 또는 `dpkg -l xz-utils` | 해당 버전 존재 시 위험 |

| systemd + sshd 연동 여부 | `systemctl status sshd` | 동작 중이면 위험 노출 |

| SSH 외부 노출 여부 | 방화벽/보안그룹 정책 확인 | 0.0.0.0:22 오픈 시 최고 위험 |

| 배포판 유형 확인 | `cat /etc/os-release` | 불안정(unstable/testing) 빌드 우선 확인 |



> **⚠️ 중요:** 본 취약점은 주로 **배포 전 테스트/불안정(unstable) 빌드** 환경에서 확인됨. 안정화(stable) 배포판 운영 환경은 상대적으로 노출 가능성이 낮으나, **버전 확인 절차는 예외 없이 수행**할 것.

## 4. 조치 및 대응 프로세스 (Action Items)



### 🔴 즉시 조치 (0~24시간 이내)



- [ ] **전 서버 XZ Utils 버전 일괄 점검** — 자산 관리 시스템 또는 원격 스크립트 활용

- [ ] **취약 버전(5.6.0 / 5.6.1) 확인 즉시 네트워크 격리** 검토

- [ ] **안전 버전(5.4.x)으로 즉시 다운그레이드** 또는 배포판 공식 보안 패치 적용



```bash

# 버전 일괄 확인 예시 (서버 다수일 경우 자동화 스크립트 활용)

xz --version

# 출력 예: xz (XZ Utils) 5.6.1 → 즉시 조치 대상



# 안전 버전 다운그레이드 (Debian/Ubuntu 계열 예시)

sudo apt install xz-utils=5.4.1-0.2

```



### 🟠 단기 조치 (24~72시간 이내)



- [ ] SSH 접근 로그 소급 분석 — 비정상 인증 시도 및 미인증 접속 흔적 확인

- [ ] 취약 기간 중 해당 서버에 대한 **침해 여부(IoC) 조사** 수행

- [ ] 외부 노출 SSH 포트에 대한 **IP 화이트리스트 적용** 또는 접근 제한 강화

- [ ] 전사 서버 대상 **취약점 스캔 결과 보고서** 보안팀 제출



### 🟡 중장기 조치 (1주 이내)



- [ ] **소프트웨어 공급망 보안 정책** 수립 또는 기존 정책 재검토

- [ ] 오픈소스 라이브러리 도입 시 **무결성 검증 프로세스(해시 검증, SBOM 관리)** 의무화 검토

- [ ] 유사 공급망 공격 대응을 위한 **모니터링 룰셋 업데이트** (SIEM, IDS)
