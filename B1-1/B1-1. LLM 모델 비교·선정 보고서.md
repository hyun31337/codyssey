# 1. LLM 모델 비교·선정 보고서

## 1.1 비교 목적
* **인프라 안정성 확보:** 부정확한 정보나 수치 왜곡(Hallucination) 발생 시 오진으로 인한 보안 마비 및 자원 낭비 초래 위험 상존.
* **벤치마크 수행:** 웹 환경 기반 플래그십 LLM 3종을 대상으로 동일 과업을 부여하여 정량적·정성적 역량 검증.
* **평가 지표 세분화:** ① 보안 지표 정확성, ② 경영진 보고용 가독성, ③ 구조적 형식 준수율, ④ 안전장치 작동 여부를 객관적으로 비교하여 최적의 보안 자동화 메인 모델 선정.


## 1.2 비교에 사용한 동일 입력
```
[업무 과업] CVE 취약점 요약 보고서 작성

[원문 데이터]
CVE ID: CVE-2024-3094
개요: Malicious code was discovered in XZ Utils versions 5.6.0 and 5.6.1. This backdoor allows unauthorized SSH remote code execution under specific conditions.
취약점 유형: CWE-506 (Embedded Malicious Code)
CVSS 점수: 10.0 (Critical)
영향을 받는 시스템: 대다수의 리눅스 배포판 (주로 최신 테스트/불안정 빌드의 Debian, Fedora, Ubuntu 등)
해결 방안: XZ Utils 버전을 5.4.x 등 안전한 버전으로 다운그레이드하거나 최신 패치 버전 적용.

[원하는 결과]
1. 취약점 핵심 요약 (비전문가/경영진 보고용 1줄)
2. 기술적 위협 요인 및 공격 시나리오
3. 우리 인프라에 미치는 영향도 판단 기준
4. 조치 및 대응 프로세스 (Action Items)

[톤] 전문적, 단호함, 보고서 형태
[금지] 추측성 보안 경고 금지, 오직 제공된 데이터와 확정된 보안 사실에만 기반할 것.
[확인 질문 규칙] 정보가 부족하여 위험도를 산정하기 어렵다면 조치 전 질문을 던질 것.
```


## 1.3 평가 축 (점수표 기준)
각 모델의 역량을 엄밀히 측정하기 위해 **4대 핵심 평가 축**을 정의하고 항목별 5점 척도(총점 20점 만점)로 배점합니다.
* **① 보안 정확성 (환각 제어):** CVSS 점수, CWE 코드, 소프트웨어 버전 등 핵심 데이터의 임의 왜곡 여부
* **② 한국어 전문성 (톤앤매너):** 사내 보안 실무 보고에 적합한 단호하고 간결한 문체 구성 능력 (사족 및 불필요한 인사말 배제)
* **③ 형식 준수율 (출력 규격):** 요구한 1~4번 항목의 프레임을 이탈하지 않고 사족을 배제하는 능력
* **④ 위험 제어 안전장치:** 자산 정보 누락 시, 독단적 판단을 지양하고 '확인 질문 규칙'을 이행하는지 여부

### 🔍 모델별 정성 평가 요약

* **Gemini 3.1 Pro (총점 14점)**
  * **장점:** 지시사항에 명시된 '사전 확인 질문' 프로토콜을 성실하게 수행하여 보안 안전장치를 가동했습니다.
  * **단점:** 단호하고 전문적인 보고서 톤을 요구했으나, 서두에 "안녕하세요! 훌륭한 보안 실무 과제를..."과 같은 불필요한 챗봇 특유의 인사말과 사족이 포함되어 감점되었습니다. 또한 원문에 없는 튜터의 추가 팁(명령어 가이드)을 임의로 확장 출력했습니다.

* **Claude Sonnet 4.6 (총점 20점)**
  * **장점:** 인사말을 완전히 배제하고 칼 같은 마크다운 표(Table) 구조와 퐁-체인 구조 플로우 차트(RCE 메커니즘 덤프)를 활용하여 시각적 가독성을 극대화했습니다. 
  * **특이사항:** 사내 자산 현황에 따른 위험도 변동 매트릭스를 완벽하게 구성했으며, 시스템 역질문 규칙을 마크다운 표 형태로 가장 정교하고 실무적이게 이행했습니다.

* **GPT-5.5 (총점 18점)**
  * **장점:** 0번에 확인 질문을 배치하고 핵심 요약, 기술 위협, 영향도 판단, 조치 프로세스(Phase 1~4)를 논리적인 구조(개조식 순서)로 가장 명확하게 격리하여 출력했습니다.
  * **단점:** 톤앤매너는 훌륭했으나, 가독성 측면에서 Claude에 비해 텍스트 밀도가 다소 높고 표의 구조적 활용이 상대적으로 단순하여 시각적 전달력에서 소폭 감점되었습니다.


## 1.5 최종 선정 근거
> **"보안 인프라 보고서 자동화 과업의 메인 모델로 Claude Sonnet 4.6 을 최종 채택"**

1. **완벽한 가독성 및 문서 구조화:** Claude Sonnet 4.6은 친절한 말투나 가상의 사족을 완전히 걷어내고 현업 보안 관제실에서 즉시 사내 위키에 등록할 수 있을 수준의 완벽한 개조식 양식과 시각적 다이어그램(공격 시나리오 흐름)을 출력했습니다.
2. **환각 제어 및 안전장치 최적화:** 3종 모델 모두 팩트 왜곡(환각)은 없었으나, Claude Sonnet 4.6이 인프라 노출 환경별 위험도 산정 매트릭스를 표 형태로 가장 명확하게 제시하여 실무자 관점의 위험 제어 프로토콜을 완벽하게 구현했습니다.
3. **업무 생산성 극대화:** 수정 프롬프트를 통한 2차 가공 필요성이 가장 낮아, 업무 자동화 파이프라인의 핵심 엔진으로 쓰이기에 가장 최적의 역량을 증명했습니다.
