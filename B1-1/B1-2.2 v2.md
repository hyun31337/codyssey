## 1. 경영진 요약 (Executive Summary)

> 리눅스 시스템 전반에 사용되는 **데이터 압축 라이브러리 XZ Utils**에 악의적 기여자에 의한 **공급망 백도어(Supply Chain Backdoor)**가 삽입됨. 취약 버전이 설치된 서버는 외부 공격자가 **SSH 인증을 완전 우회하여 인증 없이 원격 코드 실행(RCE)**이 가능한 상태임. CVSS 최고 점수(10.0) 부여된 최고 등급 위협임.

---

## 2. 기술적 위협 요인 분석

### 2-1. 취약점 본질
- **취약점 유형:** CWE-506 (Embedded Malicious Code) — 소프트웨어 빌드 산출물에 악성코드가 의도적으로 삽입된 공급망 침해
- **삽입 경로:** 악의적 행위자가 장기간 오픈소스 기여자 신뢰를 구축한 뒤 XZ Utils 공식 소스코드에 백도어 직접 커밋
- **활성화 조건:**
  - XZ Utils **5.6.0 또는 5.6.1** 설치
  - `systemd` 환경에서 `sshd(OpenSSH 데몬)` 연동 운영
  - 위 두 조건 동시 충족 시 백도어 활성화

### 2-2. 공격 메커니즘
- 악성 코드가 `liblzma` 라이브러리에 삽입되어 `sshd` 프로세스에 후킹됨
- 공격자가 **특수 제작된 SSH 패킷** 전송 시 인증 절차 완전 우회
- 인증 없이 **원격 임의 명령 실행(RCE)** 달성 → 서버 완전 장악 가능

### 2-3. 공격 흐름

```
[외부 공격자]
      │
      │  특수 제작 SSH 패킷 전송 (22/tcp)
      ▼
[SSH 데몬 (sshd)]
      │
      │  liblzma 백도어 후킹 → 인증 로직 우회
      ▼
[인증 없이 RCE 달성]
      │
      ▼
[서버 완전 장악]
```

## 4. 조치 및 대응 프로세스

> **전제:** Q1~Q5 확인 완료 후 해당 항목에 따라 우선순위를 재조정할 것. 자산 현황 파악 없이 아래 절차를 일괄 적용하는 것은 지양함.

### 🔴 즉시 조치 (0~24시간)
- [ ] 전 서버 XZ Utils 버전 일괄 점검 수행 (자동화 스크립트 또는 자산관리 시스템 활용)
- [ ] 취약 버전(5.6.0 / 5.6.1) 확인 즉시 — **네트워크 격리 또는 SSH 외부 접근 차단** 선행
- [ ] **안전 버전(5.4.x)으로 즉시 다운그레이드** 또는 배포판 공식 보안 패치 적용

```bash
# 버전 확인
xz --version

# 다운그레이드 예시 (Debian/Ubuntu 계열)
sudo apt install --allow-downgrades xz-utils=5.4.1-0.2

# 다운그레이드 예시 (RHEL/CentOS 계열)
sudo yum downgrade xz
```

### 🟠 단기 조치 (24~72시간)
- [ ] 취약 기간 중 해당 서버 **SSH 접근 로그 소급 분석** — 비정상 인증 시도 및 미인증 접속 흔적 확인
- [ ] **침해지표(IoC) 기반 포렌식 조사** 수행 — 취약점 공개 시점 전후 이상 행위 집중 분석
- [ ] 외부 노출 SSH 포트 대상 **IP 화이트리스트 적용** 또는 접근 제한 강화
- [ ] 전사 취약점 스캔 결과 보안팀 제출

### 🟡 중장기 조치 (1주 이내)
- [ ] **소프트웨어 공급망 보안 정책** 수립 또는 기존 정책 재검토
- [ ] 오픈소스 라이브러리 도입 시 **무결성 검증 프로세스 의무화** — 해시 검증, SBOM(소프트웨어 자재명세서) 관리 체계 도입 검토
- [ ] 유사 공급망 공격 탐지를 위한 **SIEM·IDS 룰셋 업데이트**
